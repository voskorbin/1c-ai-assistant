# Участие в проекте

Спасибо, что хотите помочь развивать проект! Issues, pull requests и предложения приветствуются.

## Как сообщить о проблеме

1. Проверьте, нет ли уже похожего issue.
2. Создайте новый issue с подробным описанием:
   - что ожидалось;
   - что произошло;
   - версия платформы 1С, ОС, версия Go (если касается шлюза);
   - шаги для воспроизведения.

## Как предложить изменение

1. Сделайте форк репозитория.
2. Создайте ветку с понятным именем, например `fix-chat-scroll` или `add-ollama-support`.
3. Внесите изменения.
4. Убедитесь, что шлюз собирается и тесты проходят:
   ```powershell
   cd gateway
   go build -o ai-assistant-go.exe .
   go test ./...
   ```
5. Для CFE желательно проверить код через BSL language server.
6. Сделайте pull request с кратким описанием изменений.

## Требования к коду

- **Go**: стандартный `gofmt`, понятные имена функций, комментарии к публичным методам.
- **1С / BSL**: следуйте существующему стилю модулей, избегайте захардкоженных значений.
- **Конфиги и секреты**: не коммитьте реальные URL, токены, пароли и локальные пути. Используйте `*.example.json`, `.env.example` и placeholder'ы.

## Лицензия

Отправляя pull request, вы соглашаетесь с тем, что ваш код будет распространяться под лицензией MIT.
