# Развёртывание

## Требования

- **1С:Предприятие 8.3** с поддержкой расширений конфигурации.
- **Веб-сервер 1С** с опубликованным HTTP-сервисом `hs/mcp/rpc`, если планируется использование MCP-сервера 1С.
- **Go 1.21+** для сборки шлюза (или готовый бинарник).
- **Node.js / Python с uv/uvx** — если используются MCP-серверы через `npx`/`uvx`.

## Загрузка / обновление CFE в базе

Расширение загружается стандартным способом из исходников в формате XML:

```cmd
"<1cv8-bin>\1cv8.exe" DESIGNER `
  /S "<1c-server>\<base-name>" `
  /N "<1c-user>" `
  /DisableStartupDialogs `
  /AutoTest `
  /LoadConfigFromFiles "<project-root>\cfe\src" `
  -Extension "ИИ_Ассистент" `
  /Out "<temp-dir>\cfe_load.log" `
  /DumpResult "<temp-dir>\cfe_load_result.txt"
```

Замените placeholder'ы на реальные значения вашего окружения.

Код результата в `cfe_load_result.txt`: `0` — успех, `1` — ошибка.

## Запуск шлюза

Реальные конфиги шлюза и секреты рекомендуется хранить вне репозитория, например в отдельной локальной папке.

Пример запуска из папки с конфигом:

```cmd
cd <local-secrets-folder>
ai-assistant-go.exe -config=config.json
```

Переменные окружения (LLM API key, пароли MCP и т.д.) можно задать в `.env`-файле или в окружении сервера.

## Настройка констант 1С

После загрузки CFE необходимо заполнить константы:

| Константа | Описание | Пример |
|-----------|----------|--------|
| `ИИ_АдресШлюза` | Адрес шлюза | `http://your-gateway-host:8000` |
| `ИИ_ИнтервалОпроса` | Интервал опроса статуса, сек | `2` |
| `ИИ_ТаймаутЗапроса` | Таймаут HTTP-соединения, сек | `60` |
| `ИИ_МаксимальныйРазмерВложения` | Максимальный размер прикрепляемого изображения, МБ | `10` |
| `ИИ_МаксимальноеКоличествоСообщенийКонтекста` | Сколько последних сообщений чата передаётся в контекст запроса | `10` |

Промпт-подсказка задаётся каждым пользователем в форме настроек (пункт меню «…» формы чата) и сохраняется в регистре `ИИ_НастройкиПользователей`.

## Роли

- `ИИ_Администратор` — изменение констант и полный доступ.
- `ИИ_Пользователь` — работа с чатом, чтение констант.

## Обновление CFE

При обновлении расширения не принимайте из основной конфигурации `ИИ_ТипыОбъектовКонтекстныхЗапросов` и `ИИ_ОткрытьЧат`, чтобы сохранить адаптацию под конкретную базу.

## Рекомендуемое развёртывание в продакшене

Для production-среды рекомендуется:
- запускать шлюз и `code-index daemon` как Windows-службы через **WinSW** или **NSSM**;
- хранить конфиги и бинарник в приватном репозитории или защищённом хранилище;
- обновлять файлы шлюза при перезапуске службы из приватного репозитория;
- выделять отдельный экземпляр шлюза с отдельным портом и конфигом для каждой информационной базы.

Подробные варианты развёртывания см. в [`gateway/deploy/README.md`](../gateway/deploy/README.md).
