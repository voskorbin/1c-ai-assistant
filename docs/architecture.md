# Архитектура

## Общая схема

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Пользователь 1С                              │
│  (тонкий клиент / веб-клиент / мобильный клиент)                     │
└──────────────────────┬──────────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────────────┐
│           Расширение конфигурации 1С (CFE) «ИИ Ассистент»            │
│  • Обработка «ИИ_Ассистент» с формой чата                           │
│  • Общий модуль «ИИ_АссистентКлиент» (вспомогательные клиентские    │
│    процедуры)                                                       │
│  • Общий модуль «ИИ_АссистентСервер» (фоновые задания и HTTP-        │
│    вызовы к шлюзу)                                                  │
│  • Константы: адрес шлюза, модель, интервал опроса, таймаут│
└──────────────────────┬──────────────────────────────────────────────┘
                        │ HTTPS / HTTP
                        ▼
┌─────────────────────────────────────────────────────────────────────┐
│                    On-premise шлюз (Go)                              │
│  • /chat            — одиночный запрос                              │
│  • /chat/stream     — запуск потоковой генерации                    │
│  • /chat/status/{id}— статус потоковой генерации                    │

└──────────────────────┬──────────────────────────────────────────────┘
                       │
         ┌─────────────┼─────────────┐
         ▼             ▼             ▼
    ┌─────────┐   ┌─────────┐   ┌──────────┐
    │   LLM   │   │  MCP    │   │  Внешние │
    │ провайдер│   │инструменты│   │  сервисы │
    └─────────┘   └─────────┘   └──────────┘
```

## Принцип «одна база — один шлюз»

Каждая информационная база 1С работает со своим экземпляром шлюза. Адрес шлюза хранится в константе расширения `ИИ_АдресШлюза`.

Это позволяет:
- независимо обновлять и масштабировать шлюзы;
- изолировать ключи и настройки LLM/MCP по базам;
- использовать одно и то же расширение (CFE) в любых конфигурациях.

## Универсальность CFE

Расширение не привязано к конкретной конфигурации:
- нет привязки к конкретной конфигурации базы;
- все настройки подключения вынесены в константы;
- для загрузки в новую базу язык `Русский` должен быть Adopted и UUID языка должен совпадать с базовым.

## Потоковая генерация

1. Клиент (форма чата) вызывает серверную функцию `ИИ_АссистентСервер.ЗапуститьЗапросКШлюзу`.
2. На сервере 1С создаётся фоновое задание и временное хранилище для результата.
3. Фоновое задание отправляет `POST /chat/stream` в шлюз и получает `request_id`.
4. Фоновое задание периодически опрашивает `GET /chat/status/{request_id}` до завершения генерации.
5. По завершении фоновое задание помещает результат (ответ или ошибку) во временное хранилище.
6. Клиент через обработчик ожидания вызывает `ИИ_АссистентСервер.ПроверитьФоновоеЗадание`, которое читает результат из временного хранилища и обновляет форму.

Интервал опроса статуса фонового задания задаётся константой `ИИ_ИнтервалОпроса`.

## Контекст объекта

Если чат открыт из формы объекта 1С, в запрос передаётся JSON с контекстом объекта (метаданные, реквизиты, текущие значения). Это позволяет задавать вопросы вида «Что означает поле ...?» или «Проверь реквизиты этого документа».

Привязка сессии к объекту реализована через определяемый тип `ИИ_ТипыОбъектовКонтекстныхЗапросов`:
- в CFE он по умолчанию имеет тип `Строка`, чтобы CFE можно было загрузить в любую конфигурацию;
- в конкретной базе его расширяют нужными справочниками и документами;
- реквизит `ИИ_СессииЧатов.ОбъектКонтекста` хранит строковое представление ссылки;
- при повторном открытии чата из формы объекта подхватывается существующая сессия.

Строковый тип в определяемом типе не обязателен: контекст объекта определяется по заполненности идентификатора, а наличие ссылочных типов позволяет общей команде `ИИ_ОткрытьЧат` автоматически появляться на формах объектов.

## Безопасность

- Доступ к шлюзу рекомендуется ограничить сетевой изоляцией: разрешён входящий трафик только с IP-адресов серверов 1С.
- Ключи LLM/MCP хранятся только на стороне шлюза.
- В продакшене между 1С и шлюзом должен использоваться HTTPS.
