# Адаптация CFE под конкретную информационную базу

CFE «ИИ Ассистент» изначально универсально и не привязано к конкретной конфигурации. При загрузке в новую базу требуется несколько адаптационных шагов.

## 1. Язык `Русский`

При первой загрузке CFE 1С спросит:

> Значение контролируемого свойства ОбъектРасширяемойКонфигурации у объекта Язык.Русский не совпадает со значением в расширяемой конфигурации

Это нормально: UUID языка `Русский` уникален для каждой конфигурации.

**Действие:** в окне загрузки нажать **Исправить**. 1С обновит UUID в CFE под текущую базу.

## 2. Определяемый тип `ИИ_ТипыОбъектовКонтекстныхЗапросов`

По умолчанию в CFE этот тип — строка длиной 500. Это позволяет CFE загружаться в любую конфигурацию, но для работы контекстных обсуждений тип нужно расширить ссылочными типами.

Чтобы чат с контекстом объекта работал со справочниками и документами базы:

1. В конфигураторе открыть `Определяемые типы → ИИ_ТипыОбъектовКонтекстныхЗапросов` в режиме редактирования CFE.
2. **Убрать** тип `Строка` из состава.
3. Добавить в состав типов нужные объекты базы, например:
   - `СправочникСсылка.Модель`
   - `СправочникСсылка.Номенклатура`
   - `ДокументСсылка.ЗаказКлиента`

> Для контекстных обсуждений в определяемом типе обязательно должны быть ссылочные типы объектов базы. Фильтрация сессий и привязка сообщений к объекту выполняются по ссылке, а не по строковому GUID.
>
> Если тип оставить только строковым, общая команда `ИИ_ОткрытьЧат` не сможет автоматически появляться на формах объектов, а контекстные сессии не будут привязываться к конкретным объектам.

## 3. Кнопка открытия чата на форме объекта

Общая команда `ИИ_ОткрытьЧат` автоматически появляется на формах объектов, входящих в `ИИ_ТипыОбъектовКонтекстныхЗапросов`. По умолчанию она попадает в панель навигации формы в виде закладки «ИИ Ассистент». При клике по ней ассистент открывается в новой вкладке, но пустая закладка навигации остаётся.

Для программного открытия чата с контекстом объекта используйте метод:

```bsl
&НаКлиенте
Процедура ОткрытьИИАссистент(Команда)
    ИИ_АссистентКлиент.ОткрытьЧатСКонтекстомОбъекта(ЭтотОбъект, Объект.Ссылка);
КонецПроцедуры
```

Если нужно разместить кнопку открытия чата в командной панели формы (а не в панели навигации) и убрать пустую закладку:

1. Заимствовать объект (справочник/документ) в CFE или в отдельное расширение.
2. Добавить на форму кнопку с обработчиком `ОткрытьИИАссистент` (см. пример выше).
3. При необходимости убрать общую команду `ИИ_ОткрытьЧат` из командного интерфейса этой формы.

## 4. Общая команда `ИИ_ОткрытьЧат`

В универсальном CFE общая команда `ИИ_ОткрытьЧат` **не имеет типа параметра** — иначе CFE нельзя загрузить в произвольную базу, где `ИИ_ТипыОбъектовКонтекстныхЗапросов` пока строковый.

После загрузки CFE в конкретную базу, когда `ИИ_ТипыОбъектовКонтекстныхЗапросов` расширен нужными справочниками/документами, в общей команде `ИИ_ОткрытьЧат` нужно установить **Тип параметра** = `ИИ_ТипыОбъектовКонтекстныхЗапросов`. Остальные настройки команды уже заданы в CFE.

После этого команда автоматически появится на формах объектов, входящих в `ИИ_ТипыОбъектовКонтекстныхЗапросов`. Команда принимает ссылку на объект и преобразует её в GUID для MCP, а JSON-контекст формируется на сервере.

> Если автоматическое появление на формах не требуется, команду можно добавить вручную:
> 1. Открыть объект метаданных (справочник/документ) в основной конфигурации.
> 2. **Правка → Командный интерфейс**.
> 3. В разделе **Общие команды** включить `ИИ_ОткрытьЧат` в нужную группу.

## 5. Константы CFE

После загрузки CFE заполнить константы:

| Константа | Описание | Пример |
|-----------|----------|--------|
| `ИИ_АдресШлюза` | URL шлюза | `http://your-gateway-host:8000` |
| `ИИ_ИнтервалОпроса` | Период опроса статуса фонового задания, сек | `2` |
| `ИИ_ТаймаутЗапроса` | Таймаут HTTP, сек | `60` |
| `ИИ_МаксимальныйРазмерВложения` | Максимальный размер прикрепляемого изображения, МБ | `10` |
| `ИИ_МаксимальноеКоличествоСообщенийКонтекста` | Сколько последних сообщений чата передаётся в контекст запроса | `10` |

Промпт-подсказка больше не является константой: каждый пользователь задаёт её в отдельной форме настроек (пункт «Промпт-подсказка» в меню «…» формы чата), и значение сохраняется персонально в регистре `ИИ_НастройкиПользователей`.

## 6. Назначение ролей

- `ИИ_Администратор` - полный доступ, включая изменение констант.
- `ИИ_Пользователь` - доступ к чату, командам и данным чата на чтение/запись; константы - только на просмотр.

## 7. Обновление CFE

При обновлении расширения в конфигураторе 1С может появиться предложение принять изменения из основной конфигурации. **Не принимайте из основной конфигурации** следующие объекты, иначе адаптация под конкретную базу будет потеряна:

- `ИИ_ТипыОбъектовКонтекстныхЗапросов` — в нём в базе настроены нужные справочники/документы.
- `ИИ_ОткрытьЧат` — в нём в базе настроен тип параметра и группа командного интерфейса.

Сохраняйте версию этих объектов из расширения (или повторите адаптацию после обновления).

## 8. Проверка

1. Открыть форму объекта, для которого настроен контекст.
2. Нажать `ИИ_ОткрытьЧат`.
3. Убедиться, что в чате создалась сессия, привязанная к объекту.
4. Задать вопрос «расскажи про эту модель» — ИИ должен понять, о каком конкретном объекте идёт речь.
